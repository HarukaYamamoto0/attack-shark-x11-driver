# Vibration Mode Change Notification

> To better understand this document, read the `README.md` located in the root of this directory first.
>
> This document is still highly incomplete due to the lack of available information, as I do not own any mouse model
> featuring this functionality for testing.

This event is only generated by mouse models equipped with a built-in vibration motor.

Whenever the user changes the vibration mode using the dedicated hardware button, the firmware immediately sends this
event to notify the host of the newly selected mode.

## Structure

| Event Code | Params1 | Params2        |
|------------|---------|----------------|
| `0x11`     | Unknown | Vibration Mode |

## Event Code

Identifies a vibration mode change notification.

This event is sent automatically by the device whenever the user cycles through the available vibration modes using the
physical vibration button.

## Params1

The purpose of this parameter is currently unknown.

All known firmware references treat this value as unused, and no behavior appears to depend on it.

## Params2

Represents the newly selected vibration mode.

Known values are:

|  Value | Description        |
|-------:|--------------------|
| `0x00` | Vibration Disabled |
| `0x01` | Standard           |
| `0x02` | Enhanced           |
| `0x03` | Explosive          |

These names originate from the official software localization files.

## Notes

Unlike most configuration changes, this event is **unsolicited**. The host does not request it; instead, the firmware
actively reports the change immediately after the user presses the vibration mode button.

The official driver simply stores the received mode internally so the user interface remains synchronized with the
device state without requiring an additional Feature Report read.
